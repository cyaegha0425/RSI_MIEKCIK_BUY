## 项目介绍

**咩咩Kick! V3.0.0** 是一款基于 Playwright + GraphQL API 的 RSI 抢船工具，采用全 API 抢购架构，不依赖 DOM 点击，实现极速抢购。

## 核心特性

### 两种输入模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **刷新拦截模式 (intercept)** | 输入搜索关键词，T-0 刷新页面，自动从 GraphQL 响应中提取 skuId 并加购 | 不知道具体 SKU ID 的商品 |
| **SKU ID 直购模式 (sku)** | 直接输入商品 SKU ID 进行 API 加购 | 已知道目标商品 SKU ID |

### 全 API 抢购架构

- **T-10 开始轮询**：SKU 模式提前 10 秒启动 API 加购轮询
- **T-0 刷新拦截**：intercept 模式在 T-0 瞬间刷新页面，GraphQL 拦截获取 skuId
- **极速结账**：信用点自动付款，NextStep x 2 + Validate 确认
- **智能回退**：API 失败时自动回退到 DOM 加购

### 关键功能

- **SKU 收藏夹**：拦截成功自动保存所有商品（名称 + skuId），支持手动添加
- **伏击模式**：提前将商品加入购物车，T-30 自动预装填，T-0 只执行验证
- **时间校准**：NTP 采样 + 密度窗口过滤 + 自动偏移计算
- **CDP 浏览器模式**：连接本地 Edge 浏览器，无自动化检测标记
- **GUI 界面**：配置窗口 + 进度窗口双界面

### 技术栈

- Python 3.8+（已测试兼容 Python 3.14）
- Playwright（CDP 模式连接本地 Edge）
- tkinter（GUI 界面）
- GraphQL API（直连 RSI 后端）

## 使用方法

### 启动程序

```bash
python RSI_MIEKCIK_BUY.py
```

程序启动后会自动打开配置窗口。

### 基本配置流程

1. **选择输入模式**
   - **刷新拦截**：输入搜索关键词（如 "Kraken"）
   - **SKU ID 直购**：直接输入商品 SKU ID

2. **设置抢购时间**
   - 格式：YYYY-MM-DD HH:MM:SS
   - 例如：2026-06-15 00:00:00

3. **点击开始**
   - 程序会自动执行登录、预热、时间校准等流程
   - 进度窗口显示各步骤状态

### 命令行参数

| 参数 | 说明 |
|------|------|
| --no-gui | 禁用 GUI，仅使用命令行模式 |
| --test | 测试模式（跳过最终付款确认） |

## 配置说明

### 输入模式配置

#### 刷新拦截模式 (intercept)

- **搜索关键词**：输入要抢购商品的名称关键词
- **排除关键词**：用英文逗号分隔，如 "Privateer,DUR" 可排除包含这些词的商品
- **工作原理**：T-0 刷新页面 → GraphQL 拦截获取 skuId → API 加购

#### SKU ID 直购模式 (sku)

- **SKU ID**：直接输入商品 SKU ID
- **价格**：手动填写商品价格（用于信用点付款）
- **工作原理**：T-10 开始 API 加购轮询，绕过页面加载

### 时间校准

程序会自动校准本地时间与 RSI 服务器时间的偏移：

- **自动校准**：发送多次 API 请求，从响应头获取服务器时间
- **密度窗口算法**：过滤异常样本，精确计算偏移值
- **手动微调**：可在高级设置中手动输入偏移值进行微调

### 高级设置

点击配置窗口的「高级设置」按钮可访问：

| 选项 | 说明 |
|------|------|
| 手动时间偏移 | 手动微调时间偏移（秒），正值为提前，负值为延后 |
| 纯手动模式 | 跳过自动校准，仅使用手动偏移 |
| 代理服务器 | 设置 HTTP 代理（如需） |

### 伏击模式

伏击模式适合提前知道商品即将开售的场景：

1. 将目标商品加入购物车
2. 程序在 T-30 自动执行预装填（应用信用点、填写地址等）
3. T-0 瞬间执行最终验证完成付款

## 抢购流程

### 正面硬刚模式（默认）

```
1. 登录 → 预热打开页面 → 提醒清空购物车
2. 时间校准（NTP采样计算偏移）
3. 等待抢购时间
4. 拦截模式：T-0刷新 → GraphQL拦截获取skuId → API加购
   SKU模式：T-10开始API加购轮询
5. API加购失败 → 回退DOM加购
6. 极速结账：信用点应用 → NextStepx2 → Validate确认
```

### 伏击模式

```
1. 登录 → 预热打开购物车 → 时间校准
2. 等待 T-30 → 自动预装填
3. 等待 T-0 → 执行最终验证
4. 付款成功/失败通知
```

## FAQ

**Q: 程序需要一直盯着吗？**

A: 不需要。程序会自动执行整个抢购流程，完成后会有通知提醒。您可以在程序运行时做其他事情。

**Q: 抢购失败怎么办？**

A: 可能的原因包括：
- 网络不稳定
- 商品已售罄
- 时间偏移设置不当

建议检查网络、重启程序后再次尝试。

**Q: 可以同时抢多个商品吗？**

A: 当前版本每次只能抢购一个目标商品。

**Q: 伏击模式和正面硬刚模式哪个更快？**

A: 伏击模式更快，因为它提前完成预装填，T-0 只执行最终验证。但伏击模式需要提前将商品加入购物车。

**Q: 如何获取商品 SKU ID？**

A: 使用拦截模式时，程序会自动从 GraphQL 响应中提取并保存。或者可在 RSI 商品页面查看网络请求中的 skuId 参数。

**Q: 时间校准有什么用？**

A: RSI 服务器时间与本地时间可能有偏差，校准后能更精确地把握 T-0 时机，提高抢购成功率。
